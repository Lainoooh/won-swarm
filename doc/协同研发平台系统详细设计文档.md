### 3.2 表结构 SQL

```sql
-- 管理员表
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Agent 表
CREATE TABLE agents (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    secret_key VARCHAR(64) UNIQUE NOT NULL,
    roles JSON NOT NULL,
    status VARCHAR(20) DEFAULT 'offline',
    current_task_id VARCHAR(36),
    last_heartbeat DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 项目表
CREATE TABLE projects (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    manager_id VARCHAR(36),
    status VARCHAR(20) DEFAULT 'planning',
    start_date DATE,
    end_date DATE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 项目 -Agent 关联表
CREATE TABLE project_agents (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    agent_id VARCHAR(36) NOT NULL,
    role VARCHAR(50) NOT NULL,
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (agent_id) REFERENCES agents(id)
);

-- 需求表
CREATE TABLE requirements (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    type VARCHAR(30) DEFAULT 'new_feature',
    priority VARCHAR(10) DEFAULT 'P2',
    status VARCHAR(20) DEFAULT 'pending',
    creator_id VARCHAR(36),
    assignee_id VARCHAR(36),
    document_ids JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 任务表
CREATE TABLE tasks (
    id VARCHAR(36) PRIMARY KEY,
    requirement_id VARCHAR(36),
    project_id VARCHAR(36) NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    type VARCHAR(30) NOT NULL,
    status VARCHAR(20) DEFAULT 'todo',
    assignee_id VARCHAR(36),
    start_date DATETIME,
    end_date DATETIME,
    estimated_hours INTEGER,
    actual_hours INTEGER,
    parent_task_id VARCHAR(36),
    dependency_ids JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (requirement_id) REFERENCES requirements(id),
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (assignee_id) REFERENCES agents(id)
);

-- 文档表
CREATE TABLE documents (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    requirement_id VARCHAR(36),
    task_id VARCHAR(36),
    name VARCHAR(200) NOT NULL,
    original_name VARCHAR(200),
    file_path VARCHAR(500) NOT NULL,
    file_type VARCHAR(50),
    file_size INTEGER,
    uploaded_by VARCHAR(36),
    version INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (requirement_id) REFERENCES requirements(id),
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);

-- 审计日志表
CREATE TABLE audit_logs (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36),
    action VARCHAR(50) NOT NULL,
    resource_type VARCHAR(30),
    resource_id VARCHAR(36),
    details JSON,
    ip_address VARCHAR(45),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.1 认证接口

#### POST /api/auth/login
管理员登录

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**响应**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 86400,
    "user": {
      "id": "uuid-xxx",
      "username": "admin"
    }
  }
}
```

Agent 注册

**请求体**
```json
{
  "secret_key": "sk-xxxxx",
  "name": "Agent-001",

}
```

**响应**
```json
{
  "code": 200,
  "data": {

    "name": "Agent-001",
```

## 六、安全设计

### 6.1 认证安全
- 密码使用 bcrypt 加密存储
- JWT Token 设置合理过期时间
- 支持 Token 刷新机制
- 登录失败次数限制（5 次/小时）

### 6.2 授权安全
- 基于 RBAC 的权限控制
- API 级别权限验证
- 资源级别权限验证

### 6.3 数据安全
- 敏感数据加密传输（HTTPS）
- 文件上传类型校验
- SQL 注入防护（ORM 参数化）
- XSS 防护（前端转义）

### 6.4 审计日志
- 记录关键操作
- 记录登录日志
- 记录异常信息

## 七、部署设计

### 7.1 Docker 部署

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=sqlite:///./app.db
      - SECRET_KEY=your-secret-key
      - ADMIN_PASSWORD=123456
    volumes:
      - ./data:/app/data
      - ./documents:/app/documents

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
```

### 7.2 端口配置

| 服务 | 默认端口 | 说明 |
|------|---------|------|
| Backend API | 8000 | FastAPI 服务 |
| Frontend | 80 | Nginx 静态服务 |
| HTTPS | 443 | 加密访问（可选） |
