## 三、认证与注册流程

### 3.1 完整流程

```
Step 1: 用户在平台创建 Agent
        ↓
        → 平台生成 API Key (AK)
        → 用户获取 AK 并配置到 OpenCLAW
        → 平台保存 Agent 记录（状态：未激活）

Step 2: OpenCLAW SKILL 启动
        ↓
        → 读取 ~/.openclaw/role-config.json
        → 检查 ~/.openclaw/platform-config.json (URL + AK)
        → 调用 POST /api/agents/register 注册

Step 3: 平台验证并颁发 Worker SK
        ↓
        → 验证 AK 有效性
        → 生成 Worker SK（运行态凭证）
        → 更新 Agent 状态为在线
        → 返回 worker_sk 和 agent_id

Step 4: OpenCLAW 保存 Worker SK
        ↓
        → 保存到 ~/.openclaw/worker-sk.json
        → 建立 WebSocket 连接

Step 5: 建立 WebSocket 连接
        ↓
        → WS /ws/agent/connect?sk=worker_sk_xxx
        → 平台验证 SK，建立长连接
        → 返回 agent_id 和 session_id

Step 6: 开始工作
        ↓
        → 平台通过 WebSocket 推送任务
        → Agent 通过 WebSocket 上报状态和心跳
```

### 3.2 凭证管理

```
┌─────────────────────────────────────────────────────────────────┐
│ 凭证类型与用途                                                   │
├──────────────────┬──────────────────┬───────────────────────────┤
│ 凭证类型         │ 用途             │ 存储位置                  │
├──────────────────┼──────────────────┼───────────────────────────┤
│ Platform AK      │ 注册时身份验证   │ ~/.openclaw/platform-config.json │
│ (用户配置)       │ (一次性的)       │ (用户通过 Web 界面获取后配置) │
├──────────────────┼──────────────────┼───────────────────────────┤
│ Worker SK        │ 运行时认证       │ ~/.openclaw/worker-sk.json │
│ (平台颁发)       │ WebSocket 连接   │ (注册后自动保存)           │
├──────────────────┼──────────────────┼───────────────────────────┤
│ role-config.json │ 角色定义和行为   │ ~/.openclaw/role-config.json │
│ (本地配置)       │ 约束             │ (用户预先配置)             │
└──────────────────┴──────────────────┴───────────────────────────┘
```

## 四、API 接口设计

### 4.1 Agent 管理接口

```
POST /api/agents/register
├── 功能：Agent 注册（AK 换 SK）
├── 请求头：Authorization: Bearer {platform_ak}
├── 请求体：
│   {
│     "agent_id": "pc2-worker",
│     "name": "PC2-项目经理",
│     "roles": ["project-manager"],
│     "capabilities": ["task-decomposition", "progress-tracking"],
│     "max_concurrent_tasks": 3
│   }
└── 响应：
    {
      "code": 200,
      "data": {
        "agent_id": "uuid-xxx",
        "worker_sk": "worker_sk_xxx",
        "register_url": "http://server:8000/ws/agent/connect"
      }
    }

POST /api/agents/login
├── 功能：Agent 登录（获取 worker_sk）
├── 请求头：Authorization: Bearer {platform_ak}
└── 响应：同 register

GET /api/agents
├── 功能：获取 Agent 列表
├── 查询参数：status, role, page, page_size
└── 响应：Agent 列表（含当前任务信息）

GET /api/agents/:id
├── 功能：获取 Agent 详情
└── 响应：Agent 详细信息（含 current_task）
```

### 4.2 WebSocket 连接

```
WS /ws/agent/connect
├── 查询参数：sk={worker_sk}
├── 连接建立响应：
│   {
│     "type": "connected",
│     "agent_id": "uuid-xxx",
│     "session_id": "session-xxx",
│     "server_time": "2026-03-21T10:00:00Z"
│   }
└── 后续消息通过 WebSocket 传输
```

### 4.3 任务管理接口

```
POST /api/tasks/:id/claim
├── 功能：领取任务
├── 请求头：Authorization: Bearer {worker_sk}
├── 请求体：{}
└── 响应：{ "code": 200, "data": { "task_id": "xxx", "status": "claimed" } }

POST /api/tasks/:id/status
├── 功能：上报任务状态
├── 请求头：Authorization: Bearer {worker_sk}
├── 请求体：
│   {
│     "status": "in_progress",
│     "progress": 50,
│     "message": "开发中..."
│   }
└── 响应：{ "code": 200 }

POST /api/tasks/:id/complete
├── 功能：完成任务
├── 请求头：Authorization: Bearer {worker_sk}
├── 请求体：
│   {
│     "status": "completed",
│     "result": {
│       "code_repository": "https://github.com/...",
│       "artifacts": [...],
│       "test_report": {...}
│     },
│     "message": "任务已完成"
│   }
└── 响应：{ "code": 200 }

GET /api/tasks/pending
├── 功能：获取待处理任务（HTTP 轮询备用方案）
├── 请求头：Authorization: Bearer {worker_sk}
└── 响应：{ "code": 200, "data": { "tasks": [...] } }
```

## 六、数据模型设计

### 6.1 核心表结构

```sql
-- 管理员表
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 管理员默认账号：admin / 123456

-- Agent 表
CREATE TABLE agents (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    platform_ak_hash VARCHAR(64) NOT NULL,  -- 平台 AK 哈希（注册用）
    worker_sk_hash VARCHAR(64),              -- Worker SK 哈希（运行用）
    roles JSON NOT NULL,                     -- 角色列表
    capabilities JSON,
    max_concurrent_tasks INTEGER DEFAULT 3,
    status VARCHAR(20) DEFAULT 'offline',    -- offline/idle/busy
    current_task_id VARCHAR(36),
    websocket_session_id VARCHAR(64),
    last_heartbeat DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 项目表
CREATE TABLE projects (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    manager_id VARCHAR(36),
    status VARCHAR(20) DEFAULT 'planning',   -- planning/in_progress/completed/archived
    start_date DATE,
    end_date DATE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 项目-Agent 关联表
CREATE TABLE project_agents (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    agent_id VARCHAR(36) NOT NULL,
    role VARCHAR(50) NOT NULL,
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (agent_id) REFERENCES agents(id)
);

-- 需求表
CREATE TABLE requirements (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    type VARCHAR(30) DEFAULT 'new_feature',  -- new_feature/change_request/fix_request/bug_fix/optimization
    priority VARCHAR(10) DEFAULT 'P2',       -- P0/P1/P2/P3
    status VARCHAR(20) DEFAULT 'pending',    -- pending/reviewing/approved/in_development/testing/completed
    creator_id VARCHAR(36),
    assignee_id VARCHAR(36),
    document_ids JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 任务表
CREATE TABLE tasks (
    id VARCHAR(36) PRIMARY KEY,
    requirement_id VARCHAR(36),
    project_id VARCHAR(36) NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    type VARCHAR(30) NOT NULL,  -- requirement/design/ui/frontend/backend/test
    status VARCHAR(20) DEFAULT 'todo',  -- todo/in_progress/blocked/review/completed
    assignee_id VARCHAR(36),
    required_roles JSON,
    start_date DATETIME,
    end_date DATETIME,
    estimated_hours INTEGER,
    actual_hours INTEGER,
    parent_task_id VARCHAR(36),
    dependency_ids JSON,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    assigned_at DATETIME,
    accepted_at DATETIME,
    completed_at DATETIME,
    result JSON,
    FOREIGN KEY (requirement_id) REFERENCES requirements(id),
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (assignee_id) REFERENCES agents(id)
);

-- 文档表
CREATE TABLE documents (
    id VARCHAR(36) PRIMARY KEY,
    project_id VARCHAR(36) NOT NULL,
    requirement_id VARCHAR(36),
    task_id VARCHAR(36),
    name VARCHAR(200) NOT NULL,
    original_name VARCHAR(200),
    file_path VARCHAR(500) NOT NULL,
    file_type VARCHAR(50),
    file_size INTEGER,
    uploaded_by VARCHAR(36),
    version INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (requirement_id) REFERENCES requirements(id),
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);

-- 审计日志表
CREATE TABLE audit_logs (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36),
    action VARCHAR(50) NOT NULL,
    resource_type VARCHAR(30),
    resource_id VARCHAR(36),
    details JSON,
    ip_address VARCHAR(45),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 八、OpenCLAW 本地配置

### 8.1 配置文件结构

```
~/.openclaw/
├── role-config.json          # 角色配置（用户预先配置）
├── platform-config.json      # 平台配置（用户配置 URL+AK）
├── worker-sk.json            # Worker SK（注册后自动保存）
└── skills/
    └── team-collaboration/
        └── SKILL.md          # 团队协同 SKILL
```

### 8.2 配置文件模板

**role-config.json：**
```json
{
  "agentId": "pc2-worker",
  "name": "PC2-项目经理",
  "roles": ["project-manager"],
  "capabilities": ["task-decomposition", "progress-tracking", "result-summary"],
  "maxConcurrentTasks": 3
}
```

**platform-config.json：**
```json
{
  "url": "http://<server>:8000",
  "ak": "sk-xxxxxxxx"
}
```

**worker-sk.json：**
```json
{
  "worker_sk": "worker_sk_xxx",
  "agent_id": "uuid-xxx",
  "registered_at": "2026-03-21T10:00:00Z"
}
```

### 8.3 OpenCLAW 启动流程

```
1. 用户配置平台信息
   openclaw config.patch '{
     "platform": {"url": "http://x.x.x.x:8000", "ak": "sk-xxx"}
   }'

2. 用户加载 SKILL
   (自动加载 skills/team-collaboration/SKILL.md)

3. SKILL 初始化
   - 读取 ~/.openclaw/role-config.json
   - 读取 ~/.openclaw/platform-config.json
   - 检查 ~/.openclaw/worker-sk.json 是否存在

4. 如果 Worker SK 不存在
   - 调用 POST /api/agents/register 注册
   - 保存返回的 worker_sk 到 ~/.openclaw/worker-sk.json

5. 建立 WebSocket 连接
   - WS /ws/agent/connect?sk=worker_sk_xxx
   - 开始心跳（30 秒间隔）

6. 监听任务消息
   - 平台推送 → 处理任务
   - 角色匹配检查
   - 创建子代理执行
   - 上报进度到平台
```
